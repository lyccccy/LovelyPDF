# 🚀 快速设置指南

## ⚠️ 重要：启用文件访问权限

本地 PDF 文件无法显示的原因是：**扩展默认没有访问本地文件的权限**。

### 解决步骤：

1. **打开扩展管理页面**
   - 在 Chrome 地址栏输入：`chrome://extensions/`
   - 按 Enter

2. **找到您的扩展**
   - 找到 "My Custom PDF Viewer" 扩展卡片

3. **进入扩展详情**
   - 点击扩展卡片上的 "详细信息" 按钮

4. **启用文件访问** ⭐ **关键步骤**
   - 向下滚动，找到 "允许访问文件网址" 选项
   - **打开这个开关** 🔄
   - 开关应该变成蓝色/启用状态

5. **刷新扩展**
   - 返回 `chrome://extensions/`
   - 点击扩展卡片上的刷新按钮 🔄

6. **测试**
   - 将任意 PDF 文件拖入 Chrome 窗口
   - 或通过菜单 "文件 → 打开文件" 选择 PDF
   - 现在应该可以正常显示内容了！

---

## 技术说明

### 为什么需要这个权限？

- Chrome 扩展默认**无法访问** `file://` 协议的资源（安全限制）
- 必须手动授予权限后，扩展才能读取本地文件
- 这是 Chrome 的安全机制，保护用户的本地文件不被恶意扩展访问

### 工作流程

```
所有 PDF 文件（本地 + 远程）
         ↓
扩展拦截
         ↓
viewer-wrapper.html
         ↓
fetch 获取文件内容（需要"允许访问文件网址"权限访问本地文件）
         ↓
转换为 blob URL（绕过 PDF.js 的跨域安全检查）
         ↓
传递给 PDF.js 查看器
         ↓
显示 PDF 内容 ✅
```

**为什么要转换为 blob URL？**
- PDF.js 有安全检查：`file origin does not match viewer's`
- 直接传递原始 URL 会被拒绝
- blob URL 是同源的，可以正常加载

没有"允许访问文件网址"权限时，本地文件的 fetch 会被 Chrome 阻止！
