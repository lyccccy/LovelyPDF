🧪 PDF 扩展测试页面

⚠️ 测试前请确保：
  1. 已在 chrome://extensions/ 中加载或刷新扩展
  2. 扩展已启用
  3. 打开扩展的 Service Worker 控制台查看日志

测试 1: 标准 PDF 链接

这些链接应该会被扩展拦截并用自定义查看器打开：

  • Mozilla PDF.js 示例 PDF
  • W3C Dummy PDF
  • Sample PDF

测试 2: 带参数的 PDF URL

测试 URL 中包含查询参数的情况：

  • PDF with query parameter

📋 预期行为

  • ✅ 点击链接后应该跳转到扩展的 PDF 查看器
  • ✅ URL 应该包含 chrome-extension://
  • ✅ Service Worker 控制台应该显示 "Redirected PDF: [url]"
  • ✅ PDF 内容应该在自定义查看器中正常显示

🐛 故障排除

如果 PDF 没有被拦截：

  1. 检查扩展是否已启用
  2. 刷新扩展（chrome://extensions/ 中点击刷新按钮）
  3. 查看 Service Worker 控制台是否有错误
  4. 尝试在新标签页中打开链接（而不是当前标签页）
  5. 检查 Chrome 是否禁用了第三方扩展拦截 PDF

已知限制：

  • Chrome 的内置 PDF 查看器可能在某些情况下优先级更高
  • 某些网站可能使用 iframe 嵌入 PDF，这种情况更难拦截
  • 需要 CORS 支持的 PDF 可能无法加载

📝 手动测试方法

如果自动拦截不工作，您可以：

  1. 复制 PDF URL
  2. 在地址栏输入：chrome-extension://[扩展ID]/pdfjs/web/viewer.html?file=[PDF_URL]
  3. 将 [扩展ID] 替换为您的扩展 ID（在 chrome://extensions/ 中查看）
  4. 将 [PDF_URL] 替换为 URL 编码后的 PDF 链接
